---

## 1. Purpose

Define rules for short-form text so the interface:
- feels consistent
- avoids ambiguity
- remains calm and professional
- scales with AI-generated UI later

---

## 2. Button Copy Rules

### Primary Actions
- Verb-first
- Neutral
- No hype

Examples:
- Start practice
- Continue
- Review answers
- See explanation
- Try again

### Secondary Actions
- Lower emphasis
- Optional actions

Examples:
- Skip for now
- View later
- Change selection

---

## 3. Tooltip Rules

Tooltips should:
- explain *why*, not *what*
- be one sentence when possible
- avoid technical jargon unless required

Examples:
- “State-specific rules vary. Select a state to include them.”
- “This question tests a core concept used across multiple exams.”

---

## 4. Error / Disabled State Copy

Disabled items must:
- remain visible
- explain the requirement calmly

Examples:
- “Select a state to unlock this section.”
- “Available after completing a practice test.”

❌ Forbidden:
- “You must…”
- “Required”
- Red/angry language

---

## 5. Feedback Copy (Correct / Incorrect)

Correct:
- “Correct.”
- “This answer follows the rule.”

Incorrect:
- “Not quite.”
- “This choice doesn’t follow the rule explained below.”

Never:
❌ “Wrong.”
❌ “You failed this question.”

---

## 6. Canon Lock Statement

Microcopy must never:
- shame
- pressure
- oversell

END OF CANON
